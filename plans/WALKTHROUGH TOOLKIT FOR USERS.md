de tool weet waar je bent in de PDCA en helpt je de volgende juiste stap te zetten.

Hieronder krijg je een concreet ontwerp voor zo’n begeleiding, toepasbaar in jouw app.

Kernidee: van “menu’s” naar een IMS-Journey

De gebruiker werkt niet meer via losse schermen (Risico’s, Controls, Scopes), maar doorloopt een IMS-Journey:

Scope → Processen → Risico’s → Behandeling → Controls → Besluiten → Review

De UI blijft hetzelfde, maar er komt een laag erbovenop die:

weet waar de gebruiker is,

weet wat ontbreekt,

en de volgende stap visueel activeert.

1. De IMS-Wizard (ruggengraat)
Concept

Een vaste link in de UI:

🧭 IMS Setup / IMS Journey

Deze opent geen formulier, maar een stappenoverzicht.

Stappenstructuur (voorbeeld)
□ 1. Scope vaststellen
□ 2. Kritieke processen bepalen
□ 3. Risico’s identificeren
□ 4. Behandelstrategie kiezen
□ 5. Controls selecteren
□ 6. Besluiten vastleggen
□ 7. Review & in-control oordeel


Elke stap:

heeft een status (❌ / 🟡 / ✅),

toont waarom hij nog niet afgerond is,

linkt direct naar het juiste scherm.

👉 Dit is geen lineaire wizard die alles blokkeert,
maar een voortgangsregisseur.

2. Inline “Next Best Step” hints (micro-coaching)
Wat dit is

Op elk bestaand scherm (Risico’s, Controls, Scopes) komt een subtiele hint:

💡 Volgende stap in het IMS: …

Voorbeelden

Op Risico-overzicht:

“Je hebt 3 risico’s zonder behandelstrategie → kies reduceren / accepteren.”

Op Controls:

“Deze control is nog niet gekoppeld aan een risico.”

Op Scopes:

“Deze scope is nog niet vastgesteld door het DT.”

👉 Geen pop-ups, geen uitlegblokken —
maar contextuele aanwijzingen.

3. Framework-toolkits (keuzemomenten begeleiden)

Hier wordt het cool.

Toolkit-principe

Wanneer een gebruiker iets moet definiëren, geef je hem:

✨ startpunt

📚 keuzebibliotheek

🧠 best practice suggesties

3.1 Scope Toolkit

Bij “Nieuwe scope”:

Links: leeg formulier
Rechts: Scope Toolkit

In de toolkit:

Veelgebruikte scopes:

“Gemeentelijke IV-dienstverlening”

“Kritieke burgerprocessen”

Checklist:

Processen ✔

Systemen ✔

Leveranciers ✔

Tip:

“Audits vragen altijd: waarom valt X buiten scope?”

3.2 Proces Toolkit

Bij “Proces toevoegen”:

Toolkit toont:

Procescategorieën (Burgerzaken, Sociaal Domein, IV)

BCM-trigger:

“Processen met RTO < 48u zijn meestal kritisch”

Snelle selectie:

“Markeer als kritisch”

“Later beoordelen”

3.3 Risico Toolkit (framework-driven)

Bij “Nieuw risico”:

Toolkit bevat:

Framework-suggesties:

BIO / ISO / NIS2 / AVG

“Veelvoorkomende risico’s”:

Ongeautoriseerde toegang

Uitval SaaS-leverancier

Impact-hulp:

“Hoog impact = DT-besluit vereist”

👉 De gebruiker kiest, niet typt.

3.4 Control Toolkit

Bij “Control toevoegen”:

Toolkit toont:

Standaardmaatregelen per framework

Status-indicatie:

“Vaak gebruikt”

“Technisch”

“Organisatorisch”

Evidence hint:

“Deze control vraagt meestal logbestanden of auditrapporten”

4. Visuele PDCA-Ring (altijd zichtbaar)
Concept

Rechtsboven (of in dashboard):

Een PDCA-ring met voortgang:

PLAN   ●●●○
DO     ●●○○
CHECK  ●○○○
ACT    ○○○○


Klik op een fase → zie:

wat af is,

wat ontbreekt,

wat geblokkeerd is.

👉 De gebruiker ziet dat ACT nog leeg is,
en begrijpt dat besluiten ontbreken.

5. “Why am I stuck?”-uitleg (anti-frustratie)

Als iets niet kan (bijv. risico niet sluiten):

Toon niet:

❌ Actie niet toegestaan

Maar:

❌ Dit risico kan niet worden afgerond
➤ Er is nog geen behandelstrategie gekozen
➤ Geen DT-besluit vastgelegd

Met directe links.

6. Management-modus vs. Bouw-modus
Slimme rolafhankelijke begeleiding

CISO / bouwer:

ziet alle hints en toolkits

Lijnmanager:

ziet alleen wat hij moet doen

DT-lid:

ziet alleen:

besluiten

in-control oordeel

risico’s > tolerantie

👉 Geen overload, wel regie.

Samengevat: wat je toevoegt (zonder alles te herbouwen)
Functionele bouwblokken

IMS Journey (stappenoverzicht)

Contextuele “Next step” hints

Toolkits per object (scope, proces, risico, control)

PDCA-voortgangsindicator

Blokkeringsuitleg met links

Rolafhankelijke guidance

Waarom dit werkt

Gebruiker leert door te doen

Je voorkomt lege systemen

PDCA wordt ervaren, niet uitgelegd

Je product wordt coach, geen archief
